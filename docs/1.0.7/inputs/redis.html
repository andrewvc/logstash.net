redis

Read events from a redis using BLPOP

For more information about redis, see http://redis.io/

Synopsis

This is what it might look like in your config file: 

input {
  redis {
    db => ... # number, default: 0
    debug => ... # boolean
    host => ... # string, default: "127.0.0.1"
    name => ... # string, default: "default"
    password => ... # password
    port => ... # number, default: 6379
    queue => ... # string (required)
    retries => ... # number, default: 5
    tags => ... # array
    timeout => ... # number, default: 5
    type => ... # string (required)
  }
}

Details

db

  • Value type is number
  • Default value is 0

The redis database number.

debug

  • Value type is boolean
  • There is no default value for this setting.

Set this to true to enable debugging on an input.

host

  • Value type is string
  • Default value is "127.0.0.1"

The hostname of your redis server.

name

  • Value type is string
  • Default value is "default"

Name is used for logging in case there are multiple instances.

password

  • Value type is password
  • There is no default value for this setting.

Password to authenticate with. There is no authentication by default.

port

  • Value type is number
  • Default value is 6379

The port to connect on.

queue (required setting)

  • Value type is string
  • There is no default value for this setting.

The name of the redis queue (we'll use BLPOP against this).

retries

  • Value type is number
  • Default value is 5

Maximum number of retries on a read before we give up.

tags

  • Value type is array
  • There is no default value for this setting.

Add any number of arbitrary tags to your event.

This can help with processing later.

timeout

  • Value type is number
  • Default value is 5

Initial connection timeout in seconds.

type (required setting)

  • Value type is string
  • There is no default value for this setting.

Label this input with a type.
