tcp

Read events over a TCP socket.

Like stdin and file inputs, each event is assumed to be one line of text.

Synopsis

This is what it might look like in your config file: 

input {
  tcp {
    data_timeout => ... # number, default: 5
    debug => ... # boolean
    host => ... # string, default: "0.0.0.0"
    port => ... # number (required)
    tags => ... # array
    type => ... # string (required)
  }
}

Details

data_timeout

  • Value type is number
  • Default value is 5

Read timeout in seconds. If a particular tcp connection is idle for more than this timeout period, we will assume it is dead and close it.

debug

  • Value type is boolean
  • There is no default value for this setting.

Set this to true to enable debugging on an input.

host

  • Value type is string
  • Default value is "0.0.0.0"

The address to listen on

port (required setting)

  • Value type is number
  • There is no default value for this setting.

the port to listen on

tags

  • Value type is array
  • There is no default value for this setting.

Add any number of arbitrary tags to your event.

This can help with processing later.

type (required setting)

  • Value type is string
  • There is no default value for this setting.

Label this input with a type.
